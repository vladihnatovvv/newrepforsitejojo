# Структура каталогу

Нижче зафіксовані категорії товарів на основі Telegram-гілок, які ви показали на скріншотах.

## Основні категорії

| Назва | Slug | Коментар |
|---|---|---|
| Футляри | `futlyary` | Для окулярів або аксесуарів, якщо це окрема товарна група |
| Хустки | `khustky` | Шарфи / хустки |
| Кепі | `kepi` | Базова кепі |
| Кепі-реглан | `kepi-reglan` | Окрема підкатегорія або окрема категорія |
| Морячка | `moryachka` | Окрема модель головного убору |
| Кепки | `kepky` | Бейсболки / кепки |
| Солома | `soloma` | Солом'яні моделі |
| Кепка-докерка | `kepka-dokerka` | Окрема модель |
| Панами | `panamy` | Панами |
| Окуляри | `okulyary` | Якщо це окрема товарна лінійка |
| SALE | `sale` | Акційні товари |
| Сумки | `sumky` | Окрема повноцінна категорія |

## Службові гілки

Ці гілки не варто робити товарними категоріями в `WooCommerce`:

| Назва | Призначення |
|---|---|
| Фото+опис ... | Робочий контент |
| ТЗ програмісту | Службова гілка |

## Як я рекомендую завести це в WooCommerce

### Варіант 1: плоска структура

Усі категорії окремо на одному рівні:

- Футляри
- Хустки
- Кепі
- Кепі-реглан
- Морячка
- Кепки
- Солома
- Кепка-докерка
- Панами

- SALE

Це найпростіше для старту.

### Варіант 2: ієрархія

Краще для магазину, якщо хочете чистішу навігацію:

- Головні убори
  - Кепі

  - Кепки

  - Футляри

## Моя рекомендація

Для сайту я б радив саме `Варіант 2`.

Причина:

- меню буде чистіше
- фільтрація буде зрозуміліша
- каталог не буде виглядати хаотично
- далі простіше масштабувати магазин

## Що ще треба буде визначити

Для кожної категорії далі бажано зафіксувати:

- чи є підкатегорії
- чи є розміри
- чи є кольори
- чи є матеріали
- чи є сезонність
- чи це товар з варіаціями, чи окремі картки
